<semantic-input>

Form inputs with voice hints, validation, and multi-modal support

Overview

The semantic-input component provides form input fields with automatic validation states, voice dictation hints, and multi-modal support.

Examples

Text Input
<semantic-input label="Full Name" placeholder="Enter your name" type="text"></semantic-input>
Use for: Text entry fields
Validation: None by default
Voice: "Type Full Name" to dictate
Valid State
<semantic-input label="Email" value="user@example.com" type="email" valid></semantic-input>
Use for: Showing successful validation
Visual: Green border
Contrast: Meets WCAG AA
Error State
<semantic-input label="Email" value="bad-email" type="email" error="Please enter a valid email"></semantic-input>
Use for: Validation errors
Visual: Red border + error message
Accessibility: Error announced to screen readers

Properties & Attributes

Attribute Type Values Default Description
label string text Label for the input
type string text | email | password | number text Input type
placeholder string text Placeholder text
value string text Current value
error string text Error message
valid boolean true | false false Show valid state
disabled boolean true | false false Disable input
