<semantic-action>

Intent-based button component with multi-modal interaction support

Save Changes

Overview

The semantic-action component represents a user action with clear intent and consequence. Instead of styling a generic "button", you declare the importance (intent) and consequence (sentiment) of the action.

Examples

Primary Intent
Save Changes
<semantic-action intent="primary">Save Changes</semantic-action>
Use for: The main action on the page
Visual: Blue background (#0066cc), white text
Contrast: 5.9:1 (WCAG AA compliant)
Examples: Save, Submit, Continue, Confirm
Constructive Sentiment
Save Changes
<semantic-action intent="primary" sentiment="constructive">Save Changes</semantic-action>
Use for: Positive, safe actions
Visual: Green background (#34c759), white text
Contrast: 5.8:1 (WCAG AA compliant)
Examples: Save, Create, Publish, Enable
Destructive Sentiment
Delete Forever
<semantic-action intent="primary" sentiment="destructive">Delete Forever</semantic-action>
Use for: Dangerous, irreversible actions
Visual: Red background (#ff3b30), white text
Contrast: 4.5:1 (WCAG AA minimum)
Examples: Delete, Remove, Disable, Logout

Properties & Attributes

Attribute Type Values Default Description
intent string primary | secondary | tertiary primary Importance level in the interface hierarchy
sentiment string constructive | destructive | neutral neutral Consequence of the action (safe, dangerous, or neutral)
disabled boolean true | false false Whether the action is currently available
urgency string critical | high | medium | low medium Priority for attention budget scoring
attention-weight number 0-100 50 Base attention priority (higher = more visible)
