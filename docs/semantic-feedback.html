<semantic-feedback>

Notifications, toasts, and alerts with urgency-based attention scoring

✅ Changes saved successfully

Overview

The semantic-feedback component displays user notifications with automatic urgency coloring and optional auto-dismiss. Feedback respects attention budget and suppresses low-priority notifications when user is busy.

Examples

Success Notification
✅ Changes saved successfully
<semantic-feedback urgency="medium" auto-dismiss>✅ Changes saved successfully</semantic-feedback>
Use for: Success confirmations, status updates
Visual: Blue border, slide-in animation
Duration: Auto-dismisses after 3 seconds
Critical Alert
❌ Error: Unable to save changes. Please try again.
<semantic-feedback urgency="critical">❌ Error: Unable to save changes. Please try again.</semantic-feedback>
Use for: Errors, critical warnings
Visual: Red border, persistent (no auto-dismiss)
Attention: Highest priority in attention budget

Properties & Attributes

Attribute Type Values Default Description
urgency string critical | high | medium | low medium Urgency level (affects color and attention priority)
type string notification | alert | toast notification Feedback presentation style
auto-dismiss boolean true | false false Automatically dismiss after delay

Events

Event Trigger Detail Description
dismiss Dismiss button clicked or auto-timeout { type: "dismiss" } Feedback dismissed event
